33 *
34 * The order interceptor guarantees that messages are received in the same order they were 
35 * sent.
36 * This interceptor works best with the ack=true setting. <br>
37 * There is no point in 
38 * using this with the replicationMode="fastasynchqueue" as this mode guarantees ordering.<BR>
39 * If you are using the mode ack=false replicationMode=pooled, and have a lot of concurrent threads,
40 * this interceptor can really slow you down, as many messages will be completely out of order
41 * and the queue might become rather large. If this is the case, then you might want to set 
42 * the value OrderInterceptor.maxQueue = 25 (meaning that we will never keep more than 25 messages in our queue)
43 * <br><b>Configuration Options</b><br>
44 * OrderInteceptor.expire=<milliseconds> - if a message arrives out of order, how long before we act on it <b>default=3000ms</b><br>
45 * OrderInteceptor.maxQueue=<max queue size> - how much can the queue grow to ensure ordering. 
46 *   This setting is useful to avoid OutOfMemoryErrors<b>default=Integer.MAX_VALUE</b><br>
47 * OrderInterceptor.forwardExpired=<boolean> - this flag tells the interceptor what to
