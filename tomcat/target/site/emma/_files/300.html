37 * <p>Title: Auto merging leader election algorithm</p>
38 *
39 * <p>Description: Implementation of a simple coordinator algorithm that not only selects a coordinator,
40 *    it also merges groups automatically when members are discovered that werent part of the 
41 *    </p>
42 * <p>This algorithm is non blocking meaning it allows for transactions while the coordination phase is going on
43 * </p>
44 * <p>This implementation is based on a home brewed algorithm that uses the AbsoluteOrder of a membership
45 * to pass a token ring of the current membership.<br>
46 * This is not the same as just using AbsoluteOrder! Consider the following scenario:<br>
47 * Nodes, A,B,C,D,E on a network, in that priority. AbsoluteOrder will only work if all
48 * nodes are receiving pings from all the other nodes. 
49 * meaning, that node{i} receives pings from node{all}-node{i}<br>
50 * but the following could happen if a multicast problem occurs.
51 * A has members {B,C,D}<br>
52 * B has members {A,C}<br>
53 * C has members {D,E}<br>
54 * D has members {A,B,C,E}<br>
55 * E has members {A,C,D}<br>
56 * Because the default Tribes membership implementation, relies on the multicast packets to 
57 * arrive at all nodes correctly, there is nothing guaranteeing that it will.<br>
58 * <br>
59 * To best explain how this algorithm works, lets take the above example:
60 * For simplicity we assume that a send operation is O(1) for all nodes, although this algorithm will work
61 * where messages overlap, as they all depend on absolute order<br>
62 * Scenario 1: A,B,C,D,E all come online at the same time
63 * Eval phase, A thinks of itself as leader, B thinks of A as leader,
64 * C thinks of itself as leader, D,E think of A as leader<br>
65 * Token phase:<br>
66 * (1) A sends out a message X{A-ldr, A-src, mbrs-A,B,C,D} to B where X is the id for the message(and the view)<br>
67 * (1) C sends out a message Y{C-ldr, C-src, mbrs-C,D,E} to D where Y is the id for the message(and the view)<br>
68 * (2) B receives X{A-ldr, A-src, mbrs-A,B,C,D}, sends X{A-ldr, A-src, mbrs-A,B,C,D} to C <br>
69 * (2) D receives Y{C-ldr, C-src, mbrs-C,D,E} D is aware of A,B, sends Y{A-ldr, C-src, mbrs-A,B,C,D,E} to E<br>
70 * (3) C receives X{A-ldr, A-src, mbrs-A,B,C,D}, sends X{A-ldr, A-src, mbrs-A,B,C,D,E} to D<br>
71 * (3) E receives Y{A-ldr, C-src, mbrs-A,B,C,D,E} sends Y{A-ldr, C-src, mbrs-A,B,C,D,E} to A<br>
72 * (4) D receives X{A-ldr, A-src, mbrs-A,B,C,D,E} sends sends X{A-ldr, A-src, mbrs-A,B,C,D,E} to A<br>
73 * (4) A receives Y{A-ldr, C-src, mbrs-A,B,C,D,E}, holds the message, add E to its list of members<br>
74 * (5) A receives X{A-ldr, A-src, mbrs-A,B,C,D,E} <br>
75 * At this point, the state looks like<br>
76 * A - {A-ldr, mbrs-A,B,C,D,E, id=X}<br>
77 * B - {A-ldr, mbrs-A,B,C,D, id=X}<br>
78 * C - {A-ldr, mbrs-A,B,C,D,E, id=X}<br>
79 * D - {A-ldr, mbrs-A,B,C,D,E, id=X}<br>
80 * E - {A-ldr, mbrs-A,B,C,D,E, id=Y}<br>
81 * <br>
82 * A message doesn't stop until it reaches its original sender, unless its dropped by a higher leader.
83 * As you can see, E still thinks the viewId=Y, which is not correct. But at this point we have 
84 * arrived at the same membership and all nodes are informed of each other.<br>
85 * To synchronize the rest we simply perform the following check at A when A receives X:<br>
86 * Original X{A-ldr, A-src, mbrs-A,B,C,D} == Arrived X{A-ldr, A-src, mbrs-A,B,C,D,E}<br>
87 * Since the condition is false, A, will resend the token, and A sends X{A-ldr, A-src, mbrs-A,B,C,D,E} to B
88 * When A receives X again, the token is complete. <br>
89 * Optionally, A can send a message X{A-ldr, A-src, mbrs-A,B,C,D,E confirmed} to A,B,C,D,E who then
90 * install and accept the view.
91 * </p>
92 * <p>
93 * Lets assume that C1 arrives, C1 has lower priority than C, but higher priority than D.<br>
94 * Lets also assume that C1 sees the following view {B,D,E}<br>
95 * C1 waits for a token to arrive. When the token arrives, the same scenario as above will happen.<br>
96 * In the scenario where C1 sees {D,E} and A,B,C can not see C1, no token will ever arrive.<br>
97 * In this case, C1 sends a Z{C1-ldr, C1-src, mbrs-C1,D,E} to D<br>
98 * D receives Z{C1-ldr, C1-src, mbrs-C1,D,E} and sends Z{A-ldr, C1-src, mbrs-A,B,C,C1,D,E} to E<br>

207                return; //already have this view installed
208            }            
209            int prio = AbsoluteOrder.comp.compare(local,others[0]);
210            MemberImpl leader = ( prio < 0 )?local:others[0];//am I the leader in my view?
211            if ( local.equals(leader) || force ) {
212                CoordinationMessage msg = createElectionMsg(local, others, leader);
213                suggestedviewId = msg.getId();
214                suggestedView = new Membership(local,AbsoluteOrder.comp,true);

216                fireInterceptorEvent(new CoordinationEvent(CoordinationEvent.EVT_PROCESS_ELECT,this,"Election, sending request"));
217                sendElectionMsg(local,others[0],msg);
218            } else {
219                try {
220                    coordMsgReceived.set(false);
221                    fireInterceptorEvent(new CoordinationEvent(CoordinationEvent.EVT_WAIT_FOR_MSG,this,"Election, waiting for request"));

223                }catch ( InterruptedException x ) {
224                    Thread.currentThread().interrupted();
225                }
226                if ( suggestedviewId == null && (!coordMsgReceived.get())) {
227                    //no message arrived, send the coord msg
228//                    fireInterceptorEvent(new CoordinationEvent(CoordinationEvent.EVT_WAIT_FOR_MSG,this,"Election, waiting timed out."));
229//                    startElection(true);
230                    fireInterceptorEvent(new CoordinationEvent(CoordinationEvent.EVT_ELECT_ABANDONED,this,"Election abandoned, waiting timed out."));
231                } else {
232                    fireInterceptorEvent(new CoordinationEvent(CoordinationEvent.EVT_ELECT_ABANDONED,this,"Election abandoned, received a message"));
233                }
234            }//end if

691        public void parse() {
692            //header
693            int offset = 16;
694            //leader
695            int ldrLen = buf.toInt(buf.getBytesDirect(),offset);
696            offset += 4;
697            byte[] ldr = new byte[ldrLen];
698            System.arraycopy(buf.getBytesDirect(),offset,ldr,0,ldrLen);
699            leader = MemberImpl.getMember(ldr);
700            offset += ldrLen;
701            //source
702            int srcLen = buf.toInt(buf.getBytesDirect(),offset);
703            offset += 4;
704            byte[] src = new byte[srcLen];
705            System.arraycopy(buf.getBytesDirect(),offset,src,0,srcLen);
706            source = MemberImpl.getMember(src);
707            offset += srcLen;
708            //view
709            int mbrCount = buf.toInt(buf.getBytesDirect(),offset);
710            offset += 4;
711            view = new MemberImpl[mbrCount];
712            for (int i=0; i<view.length; i++ ) {
713                int mbrLen = buf.toInt(buf.getBytesDirect(),offset);
714                offset += 4;
715                byte[] mbr = new byte[mbrLen];
716                System.arraycopy(buf.getBytesDirect(), offset, mbr, 0, mbrLen);
717                view[i] = MemberImpl.getMember(mbr);
718                offset += mbrLen;
719            }
720            //id
